Common terms

Short definitions for readers who should not need to decode repository terminology.

Main flow
A complete user or Admin journey with a clear beginning, outcome and next destination.
Subflow
Reusable behavior used inside one or more main flows, such as OTP, media upload or permission recovery.
Contract
The agreed request, response, event or error shape that connected systems must follow.
OpenAPI
A machine-readable description of the current HTTP API baseline; provisional gaps still block invention.
Data model
A business record, its fields, ownership, relationships, privacy rules and persistence readiness.
Provider
An external service such as payments, maps, identity verification, messaging or monitoring.
Build plan
The approved delivery order for a team, separated into weeks and daily working outcomes.
Weekly pack
One team-specific Markdown file containing Day 1 through Day 5 and the agent prompt for each day.
Release gate
A required evidence checkpoint that must pass before an authorized Go/No-Go decision.
Canonical source
The file that must be edited because it controls its generated outputs.
Generated output
A reproducible reading or agent file that inherits authority from its canonical source and must not be edited directly.
Provisional
Usable for review and planning but not permission to invent unresolved behavior, schema or policy.
Evidence
A test result, reviewed artifact, trace, screenshot, log or approval that proves a requirement or gate.
Authority
The approved source or qualified role that controls a decision when documents differ.
